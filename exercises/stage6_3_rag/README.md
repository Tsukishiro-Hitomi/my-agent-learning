# 阶段 6.3 · RAG 检索增强 📚（动手练习）

模型没读过你的私有资料，直接问它内部知识只会瞎编。**RAG** 让它"先查资料再回答"。

## 💡 RAG 就三步（记住就懂了）
```
问题 ──▶ ① 检索 Retrieve：从你的文档里找出最相关的几段
            ② 增强 Augment ：把这几段原文塞进 prompt，当作"资料"
            ③ 生成 Generate：让模型【只依据资料】作答，并注明出处
```
和阶段 5 的 `search_files` 是近亲——那也是"在文件里找相关内容"。RAG 只是把找到的原文
**喂回给模型当上下文**。（更高级的 RAG 用"向量检索"，见 LEARNING_PLAN 里的说明；这一关先用最朴素的
关键字检索打通，思路一模一样。）

## 练习对象
`docs/` 里三份"云雀 CDN"内部资料——**里面的数字都是我编的**（并发 3200、年费 128000…），
模型不可能提前知道。所以它要是答对了，就证明你的检索真的把资料喂进去了。

## 你要做的（Part A / B / C）
- **A** `retrieve(query)`：从 `docs/` 捞出最相关的最多 top_k 段（带来源文件名）。
  打分用最朴素的启发式即可（比如 query 里的字有多少出现在这段里）。切段的 `_load_chunks()` 已给好。
- **B** `answer_with_rag(question)`：把上面三步串起来（检索 → 塞进 prompt → 调一次模型）。
- **C** `SYSTEM_PROMPT`：约束模型"只依据资料回答、资料没有就说不知道、注明出处"。

然后 `./run.sh 6.3`（先离线不花钱，全过再跑在线）。

## 通关标准
**离线**：问"并发/年费/刷新缓存"能分别捞到 product/pricing/faq 里的正确片段；返回的是**子集**不是全量；
`answer_with_rag` 确实把资料塞进了 prompt；`SYSTEM_PROMPT` 写了。
**在线**：问编造的事实（并发 3200、年费 128000），它答得对并指出来源文件。

## 卡住了怎么办
`retrieve` 的核心就是"给每段打分、排序、取前几段"。`_load_chunks()` 已经把文档切好给你了。
`answer_with_rag` 就是把 `retrieve` 的结果拼进一句话 prompt 再调模型。实在卡住，让我给**一点提示**。

> 小观察：像"云雀 CDN"这种**每篇文档都出现**的词，对简单打分是干扰项（会把标题段的分数抬高）。
> 这正是真实 RAG 要用更聪明的**向量检索**的原因。评测里的问题已经避开了这个坑，你先把朴素版跑通即可。

## 🎯 做完你就理解了
"让 AI 懂你的私有/最新数据"不是靠重新训练模型，而是 **RAG：回答前先检索、把原文塞进上下文**。
这是企业里最常见的 agent 落地方式。
